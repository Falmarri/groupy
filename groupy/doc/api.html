GET /users¶

Get a list of all the users in the system. Note: this can take some time depending on the amount of results returned.

Example request:

GET /users HTTP/1.1
Host: example.com

Query Parameters:
 
  • search – Supports full lucene search syntax (not 100% tested)
  • filter – Filters object fields from search results. Comma separated list

Multiple Field Search:

GET /users?search=(dknap* or lenards) HTTP/1.1

Fuzzy search:

GET /users?search=uid:dknapp~0.8 HTTP/1.1

Search and Filter:

GET /users?search=uid:dknapp&filter=dn,cn,title HTTP/1.1
GET /users/{username}¶

Returns the user information of the user if it exists

Query Parameters:
 
  • filter – Filters object fields from search results. Comma separated list
GET /users/{username}/groups¶

The membership information of all the groups this user belongs to. The return value is a tuple (groupname, relationship_attributes). This format can be discussed and changed if needed.

Example request:

GET /users/dknapp/groups HTTP/1.1
Host: example.com

GET /groups¶

See GET /users

GET /groups/{groupname}¶

Returns the group object. See GET /users/{username}

GET /groups/{groupname}/members¶

See GET /users/{username}/groups

GET /groups/{groupname}/{username}¶

Returns the user object if that user is a member of this group. This URI is functionally equivalent to GET /users/{username} Therefore this is valid GET /groups/{groupname}/{username}/groups

POST /cypher¶

Allows full cypher access into the underlying data. Please don’t use this yet as it has full read/write access to the entire database

POST /dbinit¶

Initializes the database. THIS IS DESTRUCTIVE. It will clear ALL data in the current database and recreate it from ldap data
